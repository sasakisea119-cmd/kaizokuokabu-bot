# 社員3: いいねワーカー（like-worker）

## 役割
`target_accounts.json` のアカウントの投稿に戦略的に「いいね」を付け、存在感を示す。

## 実行タイミング
- 1日3回（morning / noon / evening）
- 各回5〜10いいね（1日合計15〜30いいね）

## いいね戦略

### 優先度
1. **HIGH**: target_accountsの最新投稿（直近6時間以内）
2. **MEDIUM**: 投資キーワード含む投稿で、いいね数がまだ少ないもの（早めに反応 = 通知で目立つ）
3. **LOW**: フォロワーの投稿（関係維持）

### タイミング戦略
- 投稿から **30分以内** にいいねするのが最も効果的（通知で埋もれにくい）
- 深夜帯（0:00-6:00 JST）はいいねしない

## 入出力
- 入力: `data/target_accounts.json`
- 出力: `data/like_history.json`

## 使用API
- **X API v2**: POST /2/users/:id/likes（いいね実行）
- **Grok x_search**: 対象アカウントの最新投稿検索

## 制約
- 同一アカウントへのいいねは1日3回まで
- 1時間あたり最大10いいね（レートリミット対策）
- X API無料プランのいいね上限に注意（月間制限あり）
- いいねログを `employees/logs/like-worker.log` に記録

## 安全装置
- KILL_SWITCH有効時は即停止
- APIエラー3連続で当日のいいね停止
